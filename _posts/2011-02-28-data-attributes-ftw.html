HTML5 has lots of hotness in the spec, but I feel like the new custom data attribute feature is not getting the love it deserves. For those who aren't familiar with it, the purpose of the feature is to be able to attach data to DOM elements without using something like jQuery.data. The syntax is like so:

Why does this rock so hard? For many reasons, but the one I find personally most valuable is for using with jQuery plugins. I recently wrote a plugin for ZURB called Reveal (I posted about it like week) and used the data-attribute to make implementing the plugin for designers dead simple. The majority of jQuery plugins require designers to:

  • Attach all the JS (usually just jQuery then the plugin itself)
  • Create a JS file or inline a script that calls the plugin within a document.ready (often times this is the challenge for designers)
  • Change any parameters to the plugin by passing them to the call to the plugin (if a designer is calling the plugin multiple times with different parameters, it also requires multiple selectors and calls)

For developers, plugins are usually easy as cake, but for a designer not really familiar with javascript (or jQuery) even getting a plugin working can be daunting. After chatting with ZURB's brilliant engineer Matt (seriously one of the smartest guys I've ever met) about custom data attributes, I decided to use it with Reveal. The result mean running the plugin requires:

  • Attach the JS (jQuery and Reveal)
  • Put the data-reveal-id on an element to call a modal (need proper markup on the page)
  • Parameters are also passed as custom data attributes (i.e. data-animationspeed="300")

Now, this might seem like a terrible idea for developers who are used to working with large apps or JS-heavy sites. This is true, which is why I also included the usual programatic way of calling the plugin, but for a designer, it makes implementing the plugin infinitely easier in my humble opinion.
